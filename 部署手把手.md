# GHT 徒步追踪 — 部署手把手（Cloudflare 版，全程免卡）

> 你之前卡在两道墙：Render 要国际卡、Supabase 组织权限受限。  
> 这条路**注册免卡、免组织权限、不休眠**，只要会点鼠标就能上线。  
> 全程不需要敲命令（除了最后一步可选的 `git push`）。

---

## 准备：本地仓库已就绪

我已经帮你 `git init` + 提交了第一个版本，关键文件：

- `index.html` ← 前台主页面（头部待填 Worker 地址）
- `worker/index.js` ← 后端（Cloudflare Worker，已写好）
- `DEPLOY.md` ← 速查清单

唯一需要你亲自做的：用**你自己的账号**登录 Cloudflare 和 GitHub（我没法替你登）。

---

## 第 1 步：把代码推到 GitHub

1. 打开 **GitHub Desktop**（没有就去 <https://desktop.github.com> 下载安装）。
2. 登录你的 GitHub 账号。
3. 菜单 **File → Add Local Repository** → 选文件夹 `ght-trek` → **Add**。
4. 右上角 **Publish repository** → 仓库名填 `ght-trek` → 去掉 "Keep this code private" 的勾（要公开才能被 Cloudflare 免费拉取，或勾上私库也可，Cloudflare 授权时选该仓库即可）→ **Publish**。
5. 浏览器会跳出来让你授权 Cloudflare 访问 GitHub，点 **Authorize**。

> 不想装 GitHub Desktop？终端也行：
>
> ```bash
> cd ght-trek
> git remote add origin https://github.com/<你>/ght-trek.git
> git push -u origin main
> ```

---

## 第 2 步：建前台（Cloudflare Pages）

1. 打开 <https://dash.cloudflare.com> → 用邮箱注册（**不用绑卡**）。
2. 左侧点 **Workers & Pages** → 右上角 **Create** → 选 **Pages** → **Connect to Git**。
3. 授权后选 `ght-trek` 仓库 → **Begin setup**。
4. 这一页这样填：
   - **Project name**：`ght-trek`
   - **Framework preset**：选 **None**（或 "Direct Upload" 之外的无框架）
   - **Build command**：**留空**（不填）
   - **Build output directory**：填 `/`
5. 点 **Save and Deploy**。
6. 等一两分钟，会给出地址 `https://ght-trek.<子域>.pages.dev` → 先记下来（此时打开会是空白/报错，因为后端地址还没填，正常）。

> 项目里现在只有单个 `index.html`，直接由 Pages 托管，不需要构建命令。  
> 如果你之前配置过 `cp index-notwo.html index.html`，请**清空 Build command**，否则会因找不到源文件而构建失败。

---

## 第 3 步：建后台（Cloudflare Worker）

1. 左侧 **Workers & Pages** → 右上角 **Create** → 选 **Worker**。
2. 名称填 `ght-api` → 先点 **Deploy**（随便部署一次占位，后面再替换代码）。
3. 进入 `ght-api` → 点 **Edit code**（或 **Quick edit**）。
4. 把本地 `ght-trek/worker/index.js` 的**全部内容**复制粘贴进编辑器，覆盖默认代码 → 按 **Ctrl/Cmd+S** 或点 **Deploy**。
5. 回到 `ght-api` 详情页 → 点 **Settings** → **Variables**：
   - **KV namespace bindings** → **Add binding**：
     - Variable name：`GHT`
     - 选一个 KV 命名空间（没有就先去 **Workers & Pages → KV** 点 **Create a namespace** 建一个，名字随意，比如 `ght-kv`）。
   - **Environment Variables** → **Add variable**（加两条）：
     - `ADMIN_PWD` = 你的上传密码，例如 `ght2026`
     - `SECRET` = 任意一长串随机字符，例如 `z8Zk3mPq9wL2vNx5tR7yU1cB4dF6hJ0`
   - 改完变量后，回到 **Settings → 最下面 Deploy** 或代码页 **Redeploy** 让配置生效。
6. 记下 Worker 地址：`https://ght-api.<子域>.workers.dev`（页面顶部就能看到）。

---

## 第 4 步：回填后端地址

1. 用编辑器打开 `ght-trek/index.html`。
2. 找到文件最上方这一行：
   ```html
   <meta name="ght-api" content="">
   ```
   改成（换成你的真实地址）：
   ```html
   <meta name="ght-api" content="https://ght-api.your-sub.workers.dev">
   ```
3. 保存。

---

## 第 5 步：重新部署前台

1. 回到 GitHub Desktop，`ght-trek` 仓库会出现一个改动（你刚改了那行 meta）。
2. 左下角填个备注如「填后端地址」，点 **Commit to main** → 再点 **Push origin**。
3. Cloudflare Pages 会自动重新部署（1–2 分钟）。

---

## 第 6 步：验证

1. 打开 `https://ght-trek.<子域>.pages.dev`。
2. 点右上角 **🔧 数据管理**，输入第 3 步设的 `ADMIN_PWD`。
3. 能登录、能选 GPX 上传、浏览器控制台（F12）无红色 CORS 报错 → **成功上线** 🎉

---

## 常见问题

- **Cloudflare 让我绑卡？** 只有开通**付费功能**才会（如 R2、高级防火墙）。我们这条路线只用 Pages + Worker + KV，**全在免费额度内，不要绑卡**，看到付费提示直接关掉即可。
- **登录提示密码错？** 检查 `ADMIN_PWD` 环境变量是否真的保存并 Redeploy 了；meta 里的 `ght-api` 地址是否填对。
- **上传后轨迹没显示？** 地图轨迹是在你浏览器里解析的，上传只是云端备份。刷新页面、重新上传一次即可。
- **想换密码？** 去 Worker → Settings → Variables 改 `ADMIN_PWD`（同时改 `SECRET` 让旧 token 失效），Redeploy。
