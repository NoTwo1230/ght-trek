1. 顶部菜单：**File → Add Local Repository…**
2. 点 **Choose…**，选中：
   `/Users/nt/WorkBuddy/2026-07-01-10-23-33/ght-trek`

✅ 完成标志：右上角按钮变成 **"Push origin"** 且 0 个待推送。

## 步骤 2：建后台（Supabase，免卡）

1. 打开 https://supabase.com → **Start your project** → 用 GitHub / 邮箱注册（**全程不要求绑卡**）。
2. 控制台点 **New project**：
   - **Name**：`ght-trek`
   - **Database Password**：设一个（记一下，后面不常用）
   - **Region**：选 **Singapore**（离国内近）或 **Northeast Asia**
   - 点 **Create new project** → 等 1–2 分钟建好
3. 进入项目后，左侧栏点 **SQL Editor**（图标像 `</>`）→ 点 **New query**
4. 打开你电脑上的 `SUPABASE_SETUP.sql`（就在 `ght-trek` 文件夹里），**全选复制** → 粘贴进查询框
5. 点右上角 **Run**（▶ 图标）→ 下方出现绿色 `Success` 即建好表、桶和权限
6. 左侧栏点 **Settings → API**，复制两样备用（步骤 5 要用）：
   - **Project URL**：形如 `https://xxxx.supabase.co`
   - **anon public key**：一长串 `eyJ...`
   👉 **先复制到记事本里。**

## 步骤 3：建主人账号（这就是「数据管理」要输的密码）

1. 左侧栏点 **Authentication → Users** → 右上角 **Add user**
2. 填：
   - **Email**：`owner@ght.app`
   - **Password**：自己设一个强密码，例如 `Ght2026!Himalaya`（**记好，登录要用**）
   - 勾选 **Auto Confirm User**（自动确认，免收验证邮件）
3. 点 **Create user**。
   👉 这个邮箱 + 密码，就是前台「🔧 数据管理」里要输入的账号。

> 想换邮箱？可以，但必须三处一致：Supabase 这里的邮箱、`index-notwo.html` 里 `<meta name="supabase-email">`、登录时输入的账号。

## 步骤 4：建前台（Netlify，免卡）

1. 打开 https://app.netlify.com → **Sign up** → 选 **GitHub** → 授权登录。
2. 点 **Add new site → Import an existing project** → 选 **GitHub** → 找到 `ght-trek` → 点它。
3. **Site settings** 核对：
   - **Branch to deploy**：`main` ✅
   - **Build command**：**留空**
   - **Publish directory**：填 **`.`**
4. 点 **Deploy site** → 等几十秒，状态变 **Published**。
5. 记下前台地址（Site overview 顶部），形如
   `https://amazing-name-123456.netlify.app`
   （改名：Site settings → Change site name，可改成 `ght-trek.netlify.app`）
   👉 **复制到记事本。**

## 步骤 5：把 Supabase 接上前台（关键一步）

现在前台还不知道 Supabase 在哪，改 `index-notwo.html` 头部两行：

1. 用任意编辑器打开：
   `/Users/nt/WorkBuddy/2026-07-01-10-23-33/ght-trek/index-notwo.html`
2. 找到第 6–8 行（开头是 `<meta name="supabase-...`）：
   ```html
   <meta name="supabase-url" content="">
   <meta name="supabase-anon" content="">
   <meta name="supabase-email" content="owner@ght.app">
   ```
3. 把前两个 `content=""` 填上步骤 2 复制的值：
   ```html
   <meta name="supabase-url" content="https://xxxx.supabase.co">
   <meta name="supabase-anon" content="eyJxxxxxxxxxxxxx">
   <meta name="supabase-email" content="owner@ght.app">
   ```
4. **保存文件。**
5. 推送（Netlify 会自动重新部署）：
   - **GitHub Desktop**：选中 `ght-trek` → Summary 填 `connect supabase` → **Commit to main** → **Push origin**
   - **终端**：
     ```bash
     cd /Users/nt/WorkBuddy/2026-07-01-10-23-33/ght-trek
     git add index-notwo.html
     git commit -m "connect supabase"
     git push
     ```
6. 回 Netlify **Deploys** 页面，等新的部署 **Published**。

## 步骤 6：验证（5 分钟）

1. 打开前台地址 `https://xxxx.netlify.app`。
2. 右上角点 **🔧 数据管理**。
3. 输入步骤 3 设的主人密码 → 点解锁。
   - ✅ 成功：弹出上传区。
   - ❌ 失败：检查 `supabase-email` 与 Supabase 用户邮箱是否一致、密码是否对。
4. 上传一个 `.gpx` 文件 → 应提示上传成功；去 Supabase 后台 **Storage → gpx** 桶应能看到文件。
5. 刷新页面（或开无痕窗口）→ 配置（语言/出发日期）仍生效 = 持久化 OK。
6. 按 **F12** 看 Console → 不应有红色报错。

🎉 全部通过，网站上线了！把 `.netlify.app` 地址发给朋友，他们能看地图、看进度；只有你知道密码，能上传轨迹。

## 常见问题

**Q：Supabase 要绑卡吗？**
A：不用。免费层注册免卡，Auth + Storage（1GB）+ 数据库（500MB）都够用。

**Q：anon key 公开嵌在网页里安全吗？**
A：安全。它是公开密钥，真正写数据靠登录后的 JWT；RLS 已规定「公开读、登录才能写」。**千万别**把 `service_role key` 放进前端（那能绕过权限）。

**Q：上传的 GPX 会丢吗？**
A：不会。存在 Supabase Storage，不休眠、不随部署重置（这正是当初为「没国际卡」弃用 Render 的原因）。

**Q：忘了主人密码？**
A：Supabase 后台 → Authentication → 找到 `owner@ght.app` → 重置密码。

**Q：想改代码后再更新？**
A：改本地文件 → GitHub Desktop 里 Commit + Push（或终端 `git add . && git commit && git push`）→ Netlify 自动重新部署。

**Q：travel 途中用手机能上传吗？**
A：能。任何浏览器打开前台 → 🔧 数据管理 → 输密码 → 选 GPX → 上传，访客立刻可见（数据在 Supabase，不在你某台电脑上）。
